# AI Agent Instructions

## Purpose

This subdirectory contains the Prow Analyzer tool, which provides AI-assisted analysis of Prow CI job failures via CLI or Slack bot integration. The tool integrates with the ship-help MCP service to generate failure analysis.

## Local Conventions

**For detailed operational and configuration information, see [README.md](README.md).**

- **Dual interfaces**: CLI for one-shot analysis, Slack bot for automated monitoring
- **Concurrency control**: Semaphore pattern limits concurrent analyses to prevent MCP service overload
- **Error handling**: User-facing errors must be generic (don't expose internal errors to Slack users)
- **Testing patterns**: 
  - Channel-based synchronization in tests (avoid time.Sleep)
  - System tests expect `SHIP_HELP_MCP_TOKEN` environment variable (no filesystem token files)
- **Import path**: Use `github.com/RedHatQE/OpenShift-LP-QE--Tools/apps/prow-analyzer` (not the old `github.com/oramraz` path)

## Notes

**Key architectural decisions:**

- **MCP session management**: Analyzer initializes session once and reuses session ID for subsequent requests (stateful vs stateless tradeoff)
- **Regex-based URL extraction**: Uses capturing group to handle Slack `<url|label>` and Markdown `[label](url)` formats; stops at delimiters `|`, `)`, `>`, and whitespace
- **Mutex naming**: Use `Mtx` suffix (not `Mu`) per repository conventions
- **Logging**: Use `log/slog` (not logrus) for structured logging

**Non-obvious constraints:**

- Ship-help MCP responses use Server-Sent Events (SSE) format with `data:` prefix
- Slack Socket Mode requires both Bot Token (xoxb-) and App Token (xapp-)
- Test coverage threshold: 100% (enforced by scripts/test-coverage.sh)
