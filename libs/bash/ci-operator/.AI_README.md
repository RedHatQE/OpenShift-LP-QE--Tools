# AI Agent Instructions
## Purpose
Bash library functions for CI Operator Step Scripts. Organized by LP (Layered Product) team with shared libraries in `common/`.

## Local Conventions
- Functions use hyphenated PascalCase names with the `function` keyword (e.g., `function ExitTrap--PostProcessPrep ()`).
  - Pattern: `<Phase>--<Action>` where:
    - `<Phase>`: When/how the function is invoked (e.g., `ExitTrap`, `PreAction`, `PostTask`).
    - `<Action>`: What the function does (e.g., `PostProcessPrep`, `EnsureReqs`).
  - The file name matches the function name (e.g., `ExitTrap--PostProcessPrep.sh`).
- Scripts are designed to be sourced via `eval "$(curl ...)"` from CI Operator Step Scripts.
- Shell options `set -euxo pipefail; shopt -s inherit_errexit` are standard within function bodies.
- Subshell functions `{( ... )}` are used for trap handlers to isolate shell option changes.
- Environment variable names follow the pattern `LP_<TEAM>__<SECTION>__<NAME>`:
  - `<TEAM>`:
    - `IO`      - Interop
    - `CHAOS`   - Chaos
    - `G11N`    - Globalization
  - `<SECTION>`: Functional area (e.g., `ET_PPP` for usage by `ExitTrap--PostProcessPrep()`).
  - `<NAME>`: Free-form descriptive name (e.g., `NEW_TS_NAME`).

## Notes
- Directory structure: `<team>/common/` for shared libraries within a team scope.
  - Teams: `common` (cross-team shared), `interop`, `chaos`, `g11n`.
